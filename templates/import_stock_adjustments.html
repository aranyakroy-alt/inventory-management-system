Stock Adjustment Import

Use this feature to update stock quantities for multiple products at once using a CSV file. This is useful for:

  • Physical inventory count adjustments
  • Bulk stock corrections
  • End-of-period adjustments
  • Importing stock levels from external systems

CSV Format Requirements

Required Columns

SKU

Product SKU code (must match existing products)

New_Quantity

Target stock quantity (whole number, 0 or positive)

Optional Columns

Current_Quantity

Reference field (for your records, not used in import)

Reason

Specific reason for this adjustment

Important Notes

  • SKU Matching: SKUs must exactly match existing products
  • Transaction Logging: All changes will create transaction records
  • Unchanged Values: Products already at the specified quantity will be skipped
  • Validation: Negative quantities will be rejected

  • Decrease PROD-001 from 50 to 47 units (-3 change)

  • Increase PROD-003 from 0 to 100 units (+100 change)

Before Importing Stock Adjustments

  • Backup First: Export your current data before making bulk changes
  • Verify SKUs: Ensure all SKUs in your CSV match existing products
  • Double-Check Quantities: Review all new quantities for accuracy
  • Test Small Batch: Consider testing with a few products first
  • Audit Trail: All changes will be logged with timestamps and reasons
  • Cannot Undo: Stock adjustments cannot be automatically reversed
